# ADR-001: Раскладка проекта — `cmd/raxd` + `internal/...`

## Контекст
Spec bootstrap-cli (AC: «исходники разнесены по каталогам точки входа и внутренней логики
(`cmd/` + `internal/`); внутренние пакеты не импортируются извне модуля») требует выбрать
файловую раскладку каркаса `raxd`. Это одиночный бинарь, который в следующих задачах обрастёт
TLS/MCP/exec/service-логикой. Нужно решение, не противоречащее официальным рекомендациям Go.

## Решение
Использовать раскладку с явной точкой входа `cmd/raxd/main.go` и всей внутренней логикой в
`internal/...` (например `internal/cli`, `internal/config`, `internal/version` — конкретику
определяет architect). Публичного `pkg/` не вводить: у одиночного бинаря нет внешнего API.
- `internal/` обеспечивается компилятором Go (пакеты под `internal/` нельзя импортировать вне
  модуля) — это закрывает AC об изоляции на уровне инструмента, а не соглашения.
  → https://pkg.go.dev/cmd/go#hdr-Internal_Directories
- `cmd/<bin>` + `internal/` — конвенция, одобряемая официальной страницей Go про module layout.
  → https://go.dev/doc/modules/layout

## Альтернативы
- **Плоская раскладка (`main.go` в корне + `internal/`)**: официально допустима для одиночного
  бинаря, минималистична, но при появлении второго executable требует реструктуризации; для
  растущего raxd (сервис + CLI + сеть) `cmd/` предпочтительнее.
  → https://go.dev/doc/modules/layout
- **`cmd/` + `pkg/` + `internal/`**: `pkg/` предназначен для публичного импортируемого кода —
  у raxd его нет, поэтому `pkg/` лишний и вносит путаницу. Сам паттерн `pkg/` происходит из
  community-репозитория, который прямо заявляет «NOT an official standard».
  → https://github.com/golang-standards/project-layout

## Последствия
- Плюсы: соответствие официальной странице Go; компиляторная гарантия изоляции `internal/`;
  единая точка входа удобна для goreleaser-матрицы (`./cmd/raxd`); масштабируется под будущие задачи.
- Минусы/цена: чуть больше «церемонии», чем плоский `main.go`, для текущего минимального каркаса.
- Влияние на стек: согласуется со STACK (кросс-компиляция goreleaser таргетит путь к main-пакету)
  и не вводит новых зависимостей. → внутренний `.claude/reference/STACK.ru.md`
