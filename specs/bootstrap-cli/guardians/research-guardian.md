## Issues

- [ ] Issue 1: ADR-002 содержит блоки кода с объявлением переменных и вызовом `go build`
  - Где: `specs/bootstrap-cli/decisions/ADR-002-version-injection-ldflags.md`, раздел «Решение», строки 11-21
  - Почему: контракт research-analyst запрещает блоки исходного кода/реализации. Блок `var (version = "dev" ...)` и команда `go build -ldflags=...` представлены как часть «Решения» без пометки «иллюстрация» (в `research.md` аналогичный блок такую пометку имеет).
  - Что делать: заменить блоки в ADR-002 текстовым описанием паттерна со ссылкой (`https://pkg.go.dev/cmd/link`), либо добавить явную пометку «иллюстрация паттерна, не код продукта», либо перенести иллюстрацию в `research.md` и удалить из ADR.

- [ ] Issue 2: Источник поведения `adrg/xdg` на Darwin (Q4) — только корневой URL без раздела
  - Где: `specs/bootstrap-cli/research.md`, раздел Q4, строки 142-155 — все ссылки на `https://github.com/adrg/xdg` без якоря
  - Почему: утверждения о macOS-дефолте (`~/Library/Application Support`) и приоритете `XDG_CONFIG_HOME` — конкретные факты о поведении библиотеки; читатель не может немедленно перейти к источнику.
  - Что делать: уточнить URL до конкретного раздела README (`#usage` или аналог), где описано поведение на macOS/Darwin; если README не описывает явно — вынести в «Открытые вопросы» с пометкой «нуждается в уточнении по исходному коду».
