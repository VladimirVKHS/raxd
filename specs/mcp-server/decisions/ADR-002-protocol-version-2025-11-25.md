# ADR-002 (mcp-server): версия MCP-протокола `2025-11-25` и согласование версии

> Решение по развилке Q3 spec `mcp-server`. Статус — **proposed** (финальный выбор за architect).
> Нумерация локальна для задачи mcp-server.

## Контекст
В `initialize` сервер обязан вернуть строку `protocolVersion` и согласовать её с клиентом; при
HTTP-транспорте клиент шлёт заголовок `MCP-Protocol-Version`. Open Question Q3: какая версия
актуальна и какую точную строку использовать. Дефолт PM — ориентир `2025-11-25` (из MCP-INTEGRATION),
подтвердить актуальность и формат.

## Решение
Использовать **`protocolVersion: "2025-11-25"`** (формат — строка-дата `YYYY-MM-DD`).
- Это последняя датированная ревизия спецификации (предыдущая — `2025-06-18`); changelog 2025-11-25
  описывает изменения «since the previous revision, 2025-06-18».
  → https://modelcontextprotocol.io/specification/2025-11-25/changelog
- Согласование (lifecycle): клиент шлёт поддерживаемую версию; если сервер её поддерживает — ОБЯЗАН
  ответить той же строкой; иначе ОБЯЗАН вернуть другую поддерживаемую (SHOULD — самую свежую).
  → https://modelcontextprotocol.io/specification/2025-11-25/basic/lifecycle
- HTTP-заголовок (transports): после initialize клиент ОБЯЗАН слать `MCP-Protocol-Version: 2025-11-25`
  на всех запросах; при отсутствии заголовка сервер SHOULD считать `2025-03-26`; при невалидной/
  неподдерживаемой версии — ОБЯЗАН вернуть **400 Bad Request**.
  → https://modelcontextprotocol.io/specification/2025-11-25/basic/transports
- Совместимо с `MCP-INTEGRATION.ru.md` (ориентир 2025-11-25) и подтверждено в
  `specs/tls-transport/research.md`.

## Альтернативы
- **`2025-06-18` (предыдущая версия).** Хуже: не последняя; не получим уточнений 11-25 (например,
  правило «ошибки валидации ввода → Tool Execution Errors, а не Protocol Errors», важное для AC7).
  → https://modelcontextprotocol.io/specification/2025-11-25/changelog
- **`2025-03-26` (дефолт при отсутствии заголовка).** Используется лишь как fallback, когда клиент
  не прислал `MCP-Protocol-Version`; как ОБЪЯВЛЯЕМАЯ версия сервера устарела. → transports (URL выше)

## Последствия
- **Плюсы:** соответствие последней спеке; корректное согласование версии и поведение заголовка;
  правило ошибок 11-25 поддерживает AC7 (валидация ввода → `isError`, протокольные → JSON-RPC error).
- **Минусы / цена:** при выходе следующей датированной ревизии строку и поведение надо пересмотреть
  (research/architect); если выбран SDK (ADR-001 mcp-server) — версию по сути держит сам SDK, надо
  лишь сверить, что его дефолт = 2025-11-25.
- **Влияние на стек:** строковая константа версии в коде/конфиге; новых зависимостей не вводит.
