# ADR-003: Путь установки бинаря и политика sudo

## Контекст
spec distribution Q4 / AC9: бинарь должен оказаться в `PATH`; sudo запрашивать только когда целевой
каталог требует root, явно сообщая об этом; для пользовательского каталога — без sudo. AC9 проверяет
доступность независимо от выбора пути: после установки либо `command -v raxd` успешен, либо печатается
hint про добавление каталога в `PATH`. STACK по умолчанию называет `/usr/local/bin/raxd`.

## Решение
**Принят вариант C — авто-детект прав записи + переопределение через `--prefix`/`RAXD_PREFIX`**: по
умолчанию ставить в `/usr/local/bin`, если есть права на запись (или пользователь root); если каталог
не writable, но есть `sudo` и пользователь явно соглашается — sudo-установка с явным сообщением; иначе
fallback в `~/.local/bin` и ОБЯЗАТЕЛЬНО `hint:`, если каталог не в `$PATH`. Пользователь перекрывает
целевой каталог флагом `--prefix <dir>` или env `RAXD_PREFIX`. sudo запрашивается ТОЛЬКО при выборе
системного каталога и явно сообщается; демон НЕ запускается от root. Коды возврата зафиксированы в
plan.md (нехватка прав/нет sudo → код 4 + `error:`/`hint:`, AC6).

## Альтернативы
- **A: только `/usr/local/bin` (+ sudo)** — совпадает со STACK, обычно уже в `PATH`
  (https://refspecs.linuxfoundation.org/FHS_3.0/fhs/ch04s09.html), но требует root → менее дружелюбно
  непривилегированному пользователю.
- **B: только `~/.local/bin` (без sudo)** — как deno/rustup (без root,
  https://docs.deno.com/runtime/getting_started/installation/), но часто НЕ в `PATH` на свежем хосте →
  обязателен hint; расходится со STACK-дефолтом.
- **C: авто-детект + override** (ПРИНЯТО) — закрывает все три теста AC9 на обеих ролях (root и обычный
  пользователь). → spec AC9

## Последствия
- Плюсы: «делает правильное по умолчанию» и для root, и для обычного пользователя; покрывает AC9
  №1 (без sudo), №2 (доступность/hint), №3 (понятная ошибка при нехватке прав); сохраняет
  STACK-дефолт `/usr/local/bin` как предпочтительный.
- Минусы: чуть больше логики детекта/ветвления и тестов в install.sh.
- Стек: STACK §Установка фиксирует `/usr/local/bin/raxd` — дополнить упоминанием fallback
  `~/.local/bin` + override `--prefix`/`RAXD_PREFIX` (иначе STACK неполон).
- Открыто (Q4-detail, для security/реализации): macOS-раскладка (`/usr/local/bin` принадлежит root;
  Apple Silicon homebrew — `/opt/homebrew/bin`) — детект writable покрывает оба кейса; реальное
  поведение проверяется на живом macOS вне Docker (AC13).
