# ADR-005: macOS — поведение darwin-ветки install.sh (quarantine) и нотаризация

## Контекст
spec distribution Q7 / AC11, baseline §5: «правильная» дистрибуция на macOS = подпись + нотаризация
Apple Developer ID, НО при отсутствии сертификата ($99/год, Apple Developer Program) обязателен
минимум — снятие `com.apple.quarantine` + понятная инструкция. Реальный Gatekeeper-флоу непроверяем
в Docker (AC13).

## Решение
**Принят вариант C**: darwin-ветка install.sh выполняет **идемпотентный
`xattr -d com.apple.quarantine "$dst" 2>/dev/null || true` на установленном бинаре** (без ошибки,
если атрибута нет) И печатает короткую инструкцию `hint:` на случай блокировки Gatekeeper.
Полноценную подпись/нотаризацию (требует Apple Developer ID) вынести в отдельную будущую задачу. В
документации (tech-writer) зафиксировать ограничение проверки (AC13): реальный Gatekeeper-флоу
верифицируется только на живом macOS вне Docker; в контейнере — статическая проверка наличия
darwin-ветки (`xattr -d` + инструкция).

## Альтернативы
- **A: ничего не делать на macOS** — опирается на находку, что curl НЕ ставит карантин
  (https://developer.apple.com/forums/thread/666452, Quinn/DTS). Хуже: не покрывает кейс, когда
  пользователь получил архив браузером/AirDrop (карантин есть) → нарушает baseline §5 «минимум».
- **B: реальная подпись + нотаризация** — «правильно»
  (https://developer.apple.com/documentation/security/notarizing-macos-software-before-distribution),
  но НЕВОЗМОЖНО без Apple Developer ID (см. spec Out of Scope) и непроверяемо в текущей среде.
- **C: идемпотентный `xattr -d` + инструкция** (ПРИНЯТО) — выполняет baseline-минимум, безвреден
  (no-op без атрибута), покрывает оба кейса доставки.

## Последствия
- Плюсы: выполняет baseline §5 минимум; идемпотентность безопасна; не зависит от наличия сертификата.
- Минусы: не даёт «зелёного» Gatekeeper (warning при первом запуске возможен) — но это предел без
  Apple Developer ID. Реальный флоу проверяется только на живом macOS вне Docker (AC13).
- Находка-нюанс: при доставке именно через `curl|sh` бинарь обычно НЕ карантинится (curl не ставит
  атрибут — Apple/Quinn), поэтому `xattr -d` чаще всего no-op; держим как «пояс безопасности» для иных
  путей получения архива.
- Стек: новых зависимостей не вводит (`xattr` — системная утилита macOS).
