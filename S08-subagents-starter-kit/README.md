# Стартовый набор субагентов для Claude Code

Пакет из четырёх субагентов, которые превращают разработку фичи в
последовательный pipeline: `pm-spec` пишет техническое задание,
`architect` выбирает архитектуру и описывает модули, `implementer`
пишет код и прогоняет тесты, `reviewer` проверяет результат против
спецификации и плана без права что-либо менять. Один промпт — четыре
артефакта, разделение зон ответственности на уровне инструментов.

## Установка

1. Скопируйте папку `.claude/` в корень вашего репозитория:
   ```bash
   cp -r .claude /path/to/your/repo/
   ```
2. Откройте `CLAUDE.md` из кита. Дальше — два варианта.

   **Если в вашем репозитории ещё нет `CLAUDE.md`** — скопируйте файл целиком:
   ```bash
   cp CLAUDE.md /path/to/your/repo/CLAUDE.md
   ```

   **Если у вас уже есть свой `CLAUDE.md`** — НЕ копируйте поверх.
   Откройте оба файла рядом и перенесите к себе только блок про
   субагенты (правила 1-4 и пример вызова) из шаблона. Иначе
   `cp` молча перетрёт ваши собственные правила.
3. Проверьте, что у вас свежая версия Claude Code:
   ```bash
   claude --version
   ```
4. Откройте Claude Code в корне вашего репо:
   ```bash
   cd /path/to/your/repo && claude
   ```
5. Внутри Claude Code наберите `/agents` и убедитесь, что в списке
   видны все четыре имени — `pm-spec`, `architect`, `implementer`,
   `reviewer`.

## Первая проверка

Запустите в Claude Code такой запрос:

> Опиши, что нужно сделать, чтобы добавить экспорт активных
> пользователей в CSV для админов.

Если всё установилось правильно, главный Claude сам делегирует задачу
в `pm-spec` (это видно по сообщению о вызове субагента), а в итоге
у вас в репозитории появится файл `specs/<task-id>/spec.md` с
требованиями и acceptance criteria. Если вместо этого Claude сам
начал что-то писать без делегации — почти наверняка вы забыли
скопировать `CLAUDE.md` в корень репозитория.

## Подробнее

Два документа рядом, под разные задачи чтения:

- **`WALKTHROUGH.md`** — практический проход за 10 минут от пустого
  `mkdir` до увиденного pipeline. Hello-world CLI как минимальная задача,
  контраст «без кита один агент / с китом четыре роли», три способа
  убедиться, что субагенты живые. Откройте, если хочется **увидеть**,
  как это работает, а не только прочитать.
- **`GUIDE.md`** — полный гайд: разбор четырёх ролей, объяснение tools
  scoping, сценарий прогона на содержательной фиче, шесть типичных
  затыков. Откройте, если хочется понять, **почему** именно так
  устроено и как менять под свой стек.

## Пример прогона

Папка `specs/EXAMPLE-001/` хранит результат сквозного прогона
pipeline на задаче «экспорт активных пользователей в CSV». Там лежат
`spec.md` от `pm-spec`, `plan.md` от `architect` и `review.md` от
`reviewer` — все три артефакта, которые должны появиться у вас после
полного цикла. Откройте их до первого собственного запуска, чтобы
понимать ожидаемый уровень детализации и формат секций.
