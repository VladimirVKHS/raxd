# Release Plan: <название>

## Артефакты сборки
- Цели (4): `raxd_linux_amd64`, `raxd_linux_arm64`, `raxd_darwin_amd64`, `raxd_darwin_arm64`.
- Флаги: `CGO_ENABLED=0` (статическая сборка).
- Архивы: `.tar.gz` на каждую цель.
- Контрольные суммы: `SHA256SUMS` (генерация goreleaser).

## Install-скрипт
- Детект OS/arch: `uname -s` → {linux,darwin}, `uname -m` → {amd64,arm64}.
- Скачивание нужного архива релиза.
- Проверка: сверка `SHA256SUMS` (обязательно).
- Установка: бинарь в `/usr/local/bin/raxd` (`0755`).
- Регистрация сервиса: генерация и установка systemd unit / launchd plist (`Restart=on-failure`).
- Каркас безопасности: `set -euo pipefail`, всё тело в функции, `trap` на очистку temp.

## macOS
- Снятие quarantine: `xattr -d com.apple.quarantine <bin>` + понятная инструкция пользователю.
- Нотаризация: подпись + нотаризация как «правильный» путь дистрибуции (если есть аккаунт/сертификат).

## CI
- Этапы: build → test → release (goreleaser) на тег `v*.*.*`.
- Секреты — только через переменные окружения CI (никогда в файлах репозитория).

## Откат
- Как откатить релиз: не двигать опубликованные теги, выпустить патч `vX.Y.(Z+1)`.
- Что проверить после релиза: install.sh на чистой машине, контрольные суммы, запуск сервиса.
