# MCP Spec: <короткое название>

## Транспорт
- Тип: **Streamable HTTP поверх TLS** (НЕ stdio — `raxd` обслуживает удалённых клиентов).
- Эндпоинт: `https://<host>:<port>/mcp` (POST — запросы JSON-RPC; GET — SSE-поток сервер→клиент).
- Сессии: `Mcp-Session-Id` (stateless-дружелюбно).
- Обязательно: TLS (см. SECURITY-BASELINE), валидация заголовка `Origin`.

## Аутентификация и поток вызова
- Аутентификация по API-ключу до исполнения любого tool (напр. `Authorization: Bearer rax_live_…`).
- Каждый вызов tool проходит цепочку: **аутентификация → rate-limit → аудит → исполнение**
  (и аудит результата).
- Что происходит на отказе на каждом шаге (401/403 / 429 / запись в аудит).

## Tools
На каждый tool:

### tool `<имя>`
- Описание: что делает (1-2 строки).
- Входная схема (JSON):
  ```json
  { "command": "string", "args": ["string"], "timeout": "number (опц.)" }
  ```
- Выходная схема (JSON):
  ```json
  { "stdout": "string", "stderr": "string", "exit_code": "number" }
  ```
- Ошибки: коды/условия (auth, rate-limit, таймаут, неверный ввод, отказ исполнения).
