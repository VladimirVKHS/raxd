# UX Spec: <короткое название>

## Принципы
- Краткие принципы вывода: иерархия, читаемость, единый стиль, дружелюбные тексты, предсказуемость.

## Состояния вывода
Для каждого состояния — назначение + ASCII-макет.

### install (статус-блок установки)
```
<ASCII-макет: что печатает install-скрипт после установки — версия, путь, сервис, автор, примеры команд>
```

### status (`raxd status`)
```
<ASCII-макет: состояние демона, версия, аптайм, порт>
```

### key list (`raxd key list`)
```
<ASCII-таблица tablewriter: id | label | created | last-used>
```

### error (ошибки)
```
<ASCII-макет ошибки: что случилось + что сделать>
```

## Баннер автора
```
<ASCII-баннер raxd с обязательной строкой автора: Vladimir Kovalev, OEM TECH>
```

## Цвета и стиль (lipgloss)
- Палитра и роли (заголовок, успех, предупреждение, ошибка, приглушённый текст), стили рамок/таблиц.
- Ссылка на стек: `charmbracelet/lipgloss`, `charmbracelet/log`, `olekukonko/tablewriter`.

## Тексты команд и ошибок
- Тексты подсказок/успеха для команд (`key create`, `key delete`, `config port`, …).
- Тексты ошибок: формула «что случилось + что сделать», без жаргона/стек-трейсов.

## Доступность (NO_COLOR, узкий терминал)
- Поведение при `NO_COLOR`/`--no-color`: вывод без ANSI-цвета, читаемый.
- Поведение в узком терминале: перенос/усечение, таблицы не ломаются.
